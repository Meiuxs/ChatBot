# ChatBot 架构设计书

## 1. 架构概述

### 1.1 设计目标

- **前后端分离**：前端基于设计稿独立开发，通过 API 与后端通信
- **可扩展性**：预留多 AI 提供商、主题切换等扩展点
- **类型安全**：前端使用 TypeScript，后端使用 Python 类型提示
- **快速部署**：前端静态部署，后端 Serverless 或容器化

### 2.3 核心模块设计

#### 2.3.1 状态管理 (Pinia)

Pinia stores 定义在 `src/stores/` 下，包含 chatStore（会话与消息）、userStore（用户认证）、settingsStore（设置偏好）。各 store 通过组合式 API 定义，支持跨组件状态共享。

#### 2.3.2 API 服务层

前端通过 `services/` 层封装 API 调用，所有请求经 FastAPI 后端代理转发给 Supabase，前端不直连 Supabase。`authService` 和 `chatService` 有 localStorage 降级回退机制，后端不可用时自动降级到本地存储。

#### 2.3.3 组合式函数 (Composables)

| Composable | 功能说明 |
|-----------|---------|
| useAuth | 登录/注册/登出逻辑，管理认证令牌 |
| useChat | 消息发送、流式接收、打字机效果 |
| useSidebar | 侧边栏展开/收起，移动端响应式适配 |
| useTheme | 浅色/深色主题切换（class 策略） |
| useToast | 全局消息提示管理 |

### 3.2 AI Provider 扩展点设计

采用工厂模式设计，当前实现了 OpenAI 和 DeepSeek 两个流式调用 Provider（均通过 httpx.AsyncClient 请求 SSE 流）：

```
base.py — AIProvider 抽象基类（定义 chat/validate_api_key 接口）
openai.py — OpenAI 实现（流式解析 choices[*].delta.content）
deepseek.py — DeepSeek 实现（兼容 OpenAI API 格式）
factory.py — get_provider(name) / get_provider_for_model(model) 工厂方法
```

新增 Provider 只需继承 `AIProvider` 并在 `factory.py` 的 `_providers` 字典中注册。

### 3.4 认证机制

- 后端使用 Supabase Auth SDK 处理注册/登录
- JWT Token 验证支持本地 JWKS 验证 + 远程 Supabase Auth API 回退
- JWKS 公钥在应用启动时预热缓存
- 前端持有 access_token，通过 `Authorization: Bearer <token>` 请求后端

### 3.5 API Key 加密

- 用户 API Key 使用 Fernet 对称加密存储
- 加密密钥来源：优先使用 `API_KEY_SECRET` 环境变量，未配置时回退到 `SUPABASE_KEY`
- 解密仅在后端调用 AI API 时使用，GET /settings 接口返回空字符串

## 5. 安全设计

### 5.1 认证与授权

- 使用 Supabase Auth 进行用户认证
- JWT Token 验证用户身份（本地 JWKS + 远程回退）
- Row Level Security (RLS) 保护数据访问

### 5.2 API Key 安全

- 用户 API Key 使用 Fernet 加密存储
- 仅在调用 AI API 时解密使用
- GET /settings 接口不回传真实 API Key（返回空字符串）
- 前端 localStorage 中保存有明文 API Key（需注意 XSS 防护）

### 5.3 输入验证

- 请求参数使用 Pydantic 验证
- SQL 注入防护（使用 Supabase SDK）
- XSS 防护（前端 Markdown 渲染使用 marked 转义）

### 5.4 已知安全风险与待改进项

| 风险 | 当前状态 | 建议 |
|------|----------|------|
| XSS via v-html | MessageBubble 使用 v-html 渲染 Markdown，未做 HTML 净化 | 引入 DOMPurify |
| API Key 明文存储 | 前端 localStorage 存储明文 API Key | 仅存后端或加密存储 |
| 认证 Token 明文存储 | localStorage 存储明文 access_token | 考虑 httpOnly cookie |
| 加密密钥回退 | api_key_secret 未配置时回退到 supabase_key | 强制配置独立加密密钥 |
| 解密回退明文 | 解密失败时将密文作为明文返回 | 移除回退，解密失败应抛异常 |
| CORS 配置过宽 | allow_methods=["*"], allow_headers=["*"] | 显式列出允许的方法和头 |
| 缺少速率限制 | 所有端点无速率限制 | 添加 slowapi 或中间件 |
| 全局异常泄露 | 错误响应包含 str(exc) | 返回通用错误信息 |

| 前端框架 | Vue 3 | 渐进式框架，学习曲线平缓，Composition API 灵活 |
| 构建工具 | Vite | 开发体验好，热更新快 |
| 状态管理 | Pinia | Vue 官方推荐，轻量且 TypeScript 支持好 |
| 后端框架 | FastAPI | 高性能，自动文档，类型提示 |
| BaaS | Supabase | 开源，Auth + Database + Storage |
| 样式方案 | Tailwind CSS | 快速开发，原子化 CSS |
| 图表渲染 | Mermaid | 支持 Markdown 中的流程图、时序图等 |
| 代码高亮 | highlight.js | 成熟的语法高亮库，支持 190+ 语言 |
