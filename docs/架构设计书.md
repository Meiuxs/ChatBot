# ChatBot 架构设计书

## 1. 架构概述

### 1.1 设计目标

- **前后端分离**：前端基于设计稿独立开发，通过 API 与后端通信
- **可扩展性**：预留多 AI 提供商、主题切换等扩展点
- **类型安全**：前端使用 TypeScript，后端使用 Python 类型提示
- **快速部署**：前端静态部署，后端 Serverless 或容器化

### 2.3 核心模块设计

#### 2.3.1 状态管理 (Pinia)

Pinia stores 定义在 `src/stores/` 下，包含 chatStore（会话与消息）、userStore（用户认证）、settingsStore（设置偏好）。各 store 通过组合式 API 定义，支持跨组件状态共享。

#### 2.3.2 API 服务层

前端通过 `services/` 层封装 API 调用，所有请求经 FastAPI 后端代理转发给 Supabase，前端不直连 Supabase。API 调用有 localStorage 降级回退机制。

#### 2.3.3 组合式函数 (Composables)

| Composable | 功能说明 |
|-----------|---------|
| useAuth | 登录/注册/登出逻辑，管理认证令牌 |
| useChat | 消息发送、流式接收、打字机效果 |
| useSidebar | 侧边栏展开/收起，移动端响应式适配 |
| useTheme | 浅色/深色主题切换（class 策略） |
| useToast | 全局消息提示管理 |

### 3.1 目录结构

```

├── app/
│   ├── api/

│   │   ├── chat.py              # 聊天路由（stream）
│   │   ├── sessions.py          # 会话路由（sessions/messages/sync）
│   │   ├── deps.py              # 依赖注入（get_current_user, get_user_settings）
│   │   └── settings.py          # 用户设置路由
│   ├── core/

│   │   └── logging.py           # 日志配置
│   ├── providers/
│   │   ├── base.py              # AIProvider 抽象基类
│   │   ├── openai.py            # OpenAI 流式实现
│   │   └── factory.py           # Provider 工厂
│   ├── schemas/
│   │   ├── auth.py              # 认证请求/响应模型
│   │   ├── chat.py              # 聊天请求模型
│   │   └── settings.py          # 设置请求/响应模型

├── requirements.txt
├── Dockerfile
├── railway.json
└── supabase_schema.sql          # 数据库建表脚本
```

### 3.2 AI Provider 扩展点设计

采用工厂模式设计，当前实现了 OpenAI 流式调用（通过 httpx.AsyncClient 请求 SSE 流）：

```
base.py — AIProvider 抽象基类（定义 chat 接口）
openai.py — OpenAI 实现（流式解析 choices[*].delta.content）
factory.py — get_provider("openai") 工厂方法
```

新增 Provider 只需继承 `AIProvider` 并在 `factory.py` 中注册。

### 3.3 API 路由设计

| GET | /api/settings | 获取用户设置 |

## 4. 数据流设计

### 4.1 发送消息流程

```
用户输入消息
     │
     ▼
┌─────────────┐
│ 前端验证    │ ──空消息检查──> 提示错误
└─────────────┘
     │
     │ 有效消息
     ▼
┌─────────────┐
│ 添加到本地  │ ──显示用户消息
│ 消息列表    │
└─────────────┘
     │
     ▼
┌─────────────┐
│ 调用 API    │ POST /api/chat/stream
│ /api/chat  │
└─────────────┘
     │
     ▼
┌─────────────────────────────────────┐
│ 后端处理                            │
│  1. 验证用户认证                    │
│  2. 获取用户 API Key               │
│  3. 调用 AI Provider               │
│  4. 流式返回响应                    │
└─────────────────────────────────────┘
     │
     │ SSE 流
     ▼
┌─────────────┐
│ 前端渲染    │ ──逐字显示（打字机效果）
│ 消息内容    │
└─────────────┘
     │
     ▼
┌─────────────┐
│ 保存到      │ ──本地状态 + 数据库
│ 数据库      │
└─────────────┘
```

### 4.2 会话管理流程

```
新建会话
     │
     ▼
┌─────────────┐
│ 创建会话    │ ──本地状态更新
│ 对象        │
└─────────────┘
     │
     ▼
┌─────────────┐
│ 保存到      │ ──Supabase
│ 数据库      │    sessions 表
└─────────────┘
     │
     ▼
切换会话
     │
     ▼
┌─────────────┐
│ 更新当前    │ ──本地状态
│ 会话 ID    │
└─────────────┘
     │
     ▼
┌─────────────┐
│ 加载会话    │ ──从数据库获取
│ 消息列表    │    messages 表
└─────────────┘
```

## 5. 安全设计

### 5.1 认证与授权

- 使用 Supabase Auth 进行用户认证
- JWT Token 验证用户身份
- Row Level Security (RLS) 保护数据访问

### 5.2 API Key 安全

- 用户 API Key 加密存储
- 仅在调用 AI API 时解密使用
- 不在客户端暴露明文

### 5.3 输入验证

- 请求参数使用 Pydantic 验证
- SQL 注入防护（使用 Supabase SDK）
- XSS 防护（前端转义）

## 8. 技术选型理由

| 组件 | 选型 | 理由 |
|------|------|------|
| 前端框架 | Vue 3 | 渐进式框架，学习曲线平缓，Composition API 灵活 |

| BaaS | Supabase | 开源，Auth + Database + Storage |
