### 1.2 部署说明

| 平台 | 用途 | 说明 |
|------|------|------|
| **Vercel** | 前端部署 | Vue 3 静态资源托管，全球 CDN 加速，自动 HTTPS |

## 6. 数据库配置 (Supabase)

### 6.1 创建项目

1. 访问 [supabase.com](https://supabase.com)
2. 创建新项目
3. 记录项目 URL 和 `anon` / `service_role` 密钥

### 6.2 数据库迁移

在 Supabase SQL Editor 执行以下脚本：

#### 1. 创建表
```sql
-- 设置表
CREATE TABLE settings (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID REFERENCES auth.users(id) ON DELETE CASCADE NOT NULL,
    api_key TEXT,
    model TEXT DEFAULT 'gpt-4o',
    temperature FLOAT DEFAULT 0.7,
    max_tokens INT DEFAULT 2000,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    UNIQUE(user_id)
);

-- 会话表
CREATE TABLE sessions (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID REFERENCES auth.users(id) ON DELETE CASCADE NOT NULL,
    title TEXT NOT NULL DEFAULT '新对话',
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- 消息表
CREATE TABLE messages (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    session_id UUID REFERENCES sessions(id) ON DELETE CASCADE NOT NULL,
    role TEXT NOT NULL CHECK (role IN ('user', 'assistant')),
    content TEXT NOT NULL,
    model TEXT,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);
```

#### 2. 启用 RLS
```sql
-- settings 表 RLS
ALTER TABLE settings ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Users can only access their own settings"
ON settings FOR ALL
USING (auth.uid() = user_id);

-- sessions 表 RLS
ALTER TABLE sessions ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Users can only access their own sessions"
ON sessions FOR ALL
USING (auth.uid() = user_id);

CREATE POLICY "Users can only delete their own sessions"
ON sessions FOR DELETE
USING (auth.uid() = user_id);

-- messages 表 RLS
ALTER TABLE messages ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Users can access messages in their own sessions"
ON messages FOR ALL
USING (
    EXISTS (
        SELECT 1 FROM sessions
        WHERE sessions.id = messages.session_id
        AND sessions.user_id = auth.uid()
    )
);
```

#### 3. 创建自动更新触发器
```sql
-- 自动更新 sessions.updated_at
CREATE OR REPLACE FUNCTION update_session_timestamp()
RETURNS TRIGGER AS $$
BEGIN
    UPDATE sessions SET updated_at = NOW() WHERE id = NEW.session_id;
    RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER update_session_on_message
AFTER INSERT ON messages
FOR EACH ROW EXECUTE FUNCTION update_session_timestamp();
```

## 11. 常见问题

### 11.1 CORS 错误

**问题**：跨域请求被阻止

**解决**：
1. 检查后端 `CORS_ORIGINS` 配置
2. 确保包含前端域名
3. 重启后端服务

### 11.2 Supabase 连接失败

**问题**：无法连接到 Supabase

**解决**：
1. 检查 `SUPABASE_URL` 和 `SUPABASE_KEY`
2. 确认密钥为 Service Role 密钥
3. 检查 Supabase 项目状态

### 11.3 构建失败

**问题**：`npm run build` 失败

**解决**：
1. 检查 Node.js 版本（推荐 18+）
2. 清除缓存：`rm -rf node_modules && npm install`
3. 检查环境变量配置

### 11.4 流式响应不工作

**问题**：消息不是逐字显示

**解决**：
1. 检查 Nginx 配置是否支持 SSE
2. 确保代理服务器未禁用 chunked transfer
3. 检查前端流式处理代码
