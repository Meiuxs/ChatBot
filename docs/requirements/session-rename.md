# 会话重命名功能需求规格

## 1. 概述

支持用户自定义修改会话名称，替代当前仅由系统根据首条消息自动截取标题的机制。

## 2. 功能需求

### FR-1: 侧边栏会话列表重命名

| 项目 | 描述 |
|------|------|
| **触发方式** | 鼠标悬停会话项时显示编辑按钮（铅笔图标），点击进入编辑模式 |
| **编辑模式** | 标题文本替换为 `<input>` 输入框，自动聚焦并选中全部文本 |
| **确认操作** | 按 Enter 键 或 输入框失焦（blur） |
| **取消操作** | 按 Escape 键恢复原标题 |
| **空值处理** | 提交空标题时恢复为原标题，不做修改 |
| **视觉反馈** | 编辑中显示输入框光标，保存成功后 Toast 提示"会话已重命名" |

### FR-2: 顶部标题栏重命名

| 项目 | 描述 |
|------|------|
| **触发方式** | 点击聊天顶部的会话标题 |
| **编辑模式** | 标题变为输入框，自动聚焦并选中全文 |
| **确认/取消** | 与 FR-1 一致（Enter 确认，Escape 取消，blur 确认） |
| **同步** | 修改后侧边栏列表同步更新标题 |

### FR-3: 数据持久化

| 项目 | 描述 |
|------|------|
| **前端存储** | 修改后立即更新本地 `chatStore.sessions` |
| **后端同步** | 调用 `PUT /api/sessions/{id}` 同步到数据库 |
| **离线容错** | 后端不可用时写本地 localStorage，下次 sync 时同步 |

## 3. 交互流程

```
用户悬停会话项
  → 显示编辑(铅笔)按钮 + 删除按钮
  → 点击编辑按钮
  → 标题变为输入框，自动聚焦，文字全选
  → 用户输入新名称
  → Enter / blur → 保存并调用 API
  → Escape → 取消，恢复原标题
  → 顶部标题栏同步更新
```

## 4. 界面原型

```
┌──────────────────────────────────┐
│ 会话                         [+]
├──────────────────────────────────┤
│ 💬 用户消息自动标题... 🖊️  🗑️   │  ← 悬停显示编辑/删除
│ 💬 另一个对话...          🖊️  🗑️   │
│ ┌─────────────────────────┐      │
│ │ 输入新会话名称           │      │  ← 编辑模式
│ └─────────────────────────┘      │
└──────────────────────────────────┘
```

## 5. 非功能需求

- 编辑状态仅对当前用户可见（无协同编辑场景）
- API 调用失败不应阻塞 UI，使用降级策略（本地存储）
- 标题最大长度 100 字符，超出截断
