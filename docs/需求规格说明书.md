# ChatBot 需求规格说明书

## 1. 项目概述

### 1.1 项目名称
ChatBot - 智能对话系统

### 1.2 项目类型
前后端分离的全栈 Web 应用

### 1.3 核心功能
基于大模型 API 的对话聊天系统，支持多会话管理、Markdown 渲染、流式响应

### 1.4 目标用户
初学 Python 和大模型基础知识的开发者

### 1.5 项目范围

| 类别 | 范围 |
|------|------|
| 前端 | 基于设计稿重新开发，使用 Vue 3 + TypeScript |
| 后端 | Python + FastAPI，使用 BaaS 方案（Supabase） |
| 本期范围 | 对话功能、会话管理、基础设置 |
| 扩展预留 | 多 API 支持、主题切换、导入导出、快捷指令 |

---

## 2. 功能需求

### 2.1 对话功能

| 功能点 | 优先级 | 描述 |
|--------|--------|------|
| 发送消息 | P0 | 用户输入文本并发送 |
| Markdown 渲染 | P0 | AI 回复支持 Markdown 格式（标题、列表、代码块、表格等） |
| 代码高亮 | P0 | 代码块语法高亮显示 |
| 流式响应 | P0 | AI 回复逐字显示（打字机效果） |
| 快捷键发送 | P0 | Ctrl+Enter / Cmd+Enter 发送 |
| 加载状态 | P0 | 显示 AI 思考中的加载动画 |
| 错误提示 | P0 | API 错误、网络错误的友好提示 |

### 2.2 会话管理

| 功能点 | 优先级 | 描述 |
|--------|--------|------|
| 会话持久化 | P0 | 用户数据云端同步存储 |
| 新建会话 | P0 | 创建新的对话 |
| 删除会话 | P0 | 删除指定会话 |
| 切换会话 | P0 | 在不同会话间切换 |
| 会话列表 | P0 | 显示所有历史会话 |
| 会话标题 | P1 | 自动生成会话标题（首条消息摘要） |

### 2.3 设置功能

| 功能点 | 优先级 | 描述 |
|--------|--------|------|
| 用户认证 | P0 | 用户注册/登录系统 |
| API Key 配置 | P0 | 用户配置自己的 AI 服务 API Key |
| 模型选择 | P0 | 选择使用的 AI 模型 |
| Temperature | P1 | 调节回复随机性（0-2） |
| Max Tokens | P1 | 最大回复长度控制 |

## 4. 数据流需求

### 4.1 用户数据存储

- 用户认证信息：邮箱、密码（加密存储）
- 用户设置：API Key、默认模型、参数配置
- 会话数据：会话列表、消息历史

### 4.2 消息处理流程

```
用户发送消息 → 系统验证 → AI 服务处理 → 流式返回 → 存储记录
```

### 4.3 数据同步

- 多设备登录：会话数据实时同步
- 离线处理：网络恢复后自动同步

## 5. 验收标准

### 5.1 功能验收

| 编号 | 标准 | 验证方式 |
|------|------|----------|
| AC-01 | 用户可注册/登录系统 | 功能测试 |
| AC-02 | 用户可配置并保存 API Key | 功能测试 |
| AC-03 | 用户可发送消息并获得 AI 回复 | 功能测试 |
| AC-04 | Markdown 内容正确渲染 | 视觉测试 |
| AC-05 | 代码块有语法高亮 | 视觉测试 |
| AC-06 | 支持流式响应（打字机效果） | 手动测试 |
| AC-07 | 可新建、切换、删除会话 | 功能测试 |
| AC-08 | 刷新页面后会话历史保留 | 功能测试 |
| AC-09 | 快捷键 Ctrl+Enter 发送消息 | 手动测试 |
| AC-10 | 错误状态有明确提示 | 手动测试 |

### 5.2 非功能验收

| 编号 | 标准 | 目标值 |
|------|------|--------|
| NF-01 | 首屏加载时间 | < 2s |
| NF-02 | API 响应时间（不含 AI） | < 200ms |
| NF-03 | 移动端适配 | 响应式布局 |
| NF-04 | 浏览器兼容性 | Chrome/Firefox/Safari 最新版 |
| NF-05 | 无障碍支持 | WCAG 2.1 A 级 |

## 6. UI/UX 需求

### 6.1 布局结构

| 区域 | 功能 |
|------|------|
| 侧边栏 | 会话列表、新建会话、删除会话 |
| 主聊天区 | 消息展示、输入框 |
| 设置面板 | 用户设置、API 配置 |
| 头部 | 当前会话标题、设置入口 |

### 6.2 交互需求

| 功能 | 交互描述 |
|------|----------|
| 发送消息 | 点击发送按钮 或 Ctrl+Enter 快捷键 |
| 新建会话 | 点击侧边栏新建按钮 |
| 删除会话 | 悬停会话项显示删除按钮，点击删除 |
| 切换会话 | 点击会话项切换 |
| 打开设置 | 点击设置图标 |
| 关闭设置 | 点击关闭按钮 或 ESC 键 |

### 6.3 响应式需求

- 桌面端：侧边栏常驻，聊天区占主体
- 移动端：侧边栏隐藏，点击菜单按钮展开

---

## 7. 安全需求

### 7.1 认证安全

- 用户密码加密存储
- 会话 Token 过期机制
- 敏感操作二次验证

### 7.2 数据安全

- 用户 API Key 加密存储
- API Key 仅用于 AI 服务调用，不返回前端
- 用户数据隔离（仅自己能访问）

### 7.3 输入安全

- 用户输入内容转义
- 防止 XSS 攻击
- API 请求参数验证

---

## 8. 风险与约束

### 8.1 项目约束

| 约束 | 说明 |
|------|------|
| 前端基于设计稿 | 保持 UI 与设计稿一致 |
| 技术栈 | Vue 3 + TypeScript / Python + FastAPI |
| BaaS 方案 | 使用 Supabase 减少后端开发 |

### 8.2 技术风险

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| AI API 成本不可控 | 高 | 实现用量监控和限制 |
| 第三方服务稳定性 | 中 | 备用方案 |
| 流式响应兼容性 | 低 | 多浏览器测试 |

---

## 9. 术语表

| 术语 | 说明 |
|------|------|
| BaaS | Backend as a Service，后端即服务 |
| 流式响应 | Server-Sent Events，数据以流式方式传输 |
| Markdown | 轻量级标记语言 |
| API Key | 用于验证 AI 服务身份的密钥 |
| 会话 | 用户与 AI 的一次完整对话上下文 |
| RLS | Row Level Security，行级安全策略 |
